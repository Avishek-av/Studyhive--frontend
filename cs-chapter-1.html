Database Management System

A Database Management System (DBMS) is software that manages databases, allowing users to store, retrieve, update, and manage data efficiently. It provides an interface between the database and end users or application programs.

Introduction to DBMS

Database Management System (DBMS)

A DBMS is a collection of programs that enables users to create, maintain, and manipulate databases. It acts as an interface between the database and various application programs.

Database is an organized collection of related data that is stored electronically in a computer system. Data is typically organized to model relevant aspects of reality in a way that supports processes requiring this information.

Key Components of DBMS:

  • Hardware: Physical devices like computers, storage devices, and network components
  • Software: The DBMS software itself and the operating system
  • Data: The actual information stored in the database
  • Procedures: Instructions and rules for using the DBMS
  • Database Access Language: Typically SQL (Structured Query Language)
Real-World Examples

1. Banking System: Manages customer accounts, transactions, loans

2. University System: Stores student records, courses, grades

3. Airline Reservation: Manages flights, bookings, passengers

4. Hospital Management: Patient records, appointments, medical history

Advantages of DBMS:

  • Data Redundancy Control: Minimizes duplicate data
  • Data Consistency: Ensures accurate and consistent data
  • Data Security: Provides access control and security features
  • Data Integrity: Maintains accuracy and validity of data
  • Data Sharing: Allows multiple users to access data simultaneously
  • Backup and Recovery: Provides mechanisms for data backup and recovery
  • Data Independence: Changes in database structure don't affect applications

Data Models

A data model is a collection of concepts for describing data, data relationships, data semantics, and consistency constraints. It provides a way to describe the design of a database at different levels of abstraction.

1. Hierarchical Model

This model organizes data in a tree-like structure where each record has a single parent but can have multiple children. It uses parent-child relationships.

Characteristics of Hierarchical Model
  • One-to-many relationships
  • Parent-child relationship structure
  • Records are organized as trees
  • Root record has no parent
  • Child records have exactly one parent

2. Network Model

This model allows each record to have multiple parent and child records, forming a network-like structure. It's an extension of the hierarchical model.

Characteristics of Network Model
  • Many-to-many relationships
  • Records can have multiple parents
  • More flexible than hierarchical model
  • Uses pointers to represent relationships
  • Complex data structures

3. Relational Model

This model represents data as relations (tables) consisting of rows (tuples) and columns (attributes). Developed by E.F. Codd in 1970.

Key Concepts of Relational Model
  • Relation: A table with rows and columns
  • Tuple: A single row in a table
  • Attribute: A column in a table
  • Domain: Set of allowable values for an attribute
  • Key: Uniquely identifies a tuple
  • Foreign Key: Links two relations together

DBMS Architecture

DBMS architecture refers to the design and structure of a database system. The most common architecture is the three-level architecture proposed by the ANSI/SPARC committee.

Three-Level Architecture

1. External Level (View Level)

This is the highest level of abstraction that describes part of the database relevant to a particular user group. Each user has their own view of the database.

Example: A student sees only their grades, while a teacher sees all students' grades.

2. Conceptual Level (Logical Level)

This level describes what data is stored in the database and the relationships among the data. It represents the entire database structure without showing implementation details.

Example: Database schema showing tables, attributes, and relationships.

3. Internal Level (Physical Level)

This is the lowest level that describes how the data is physically stored in the storage devices. It deals with data structures, file organization, and access methods.

Example: Storage details like B-trees, hash tables, and indexing methods.

Normalization

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves decomposing tables to eliminate undesirable characteristics.

Goals of Normalization
  • Eliminate redundant data
  • Ensure data dependencies make sense
  • Reduce insertion, update, and deletion anomalies
  • Improve data integrity
  • Simplify database structure

Normal Forms

1. First Normal Form (1NF)

A table is in 1NF if:

  • Each column contains atomic (indivisible) values
  • Each column contains values of a single type
  • Each column has a unique name
  • The order of data doesn't matter
2. Second Normal Form (2NF)

A table is in 2NF if:

  • It is in 1NF
  • All non-key attributes are fully functionally dependent on the primary key
  • No partial dependency exists
3. Third Normal Form (3NF)

A table is in 3NF if:

  • It is in 2NF
  • No transitive dependency exists
  • All non-key attributes depend only on the primary key
